Functional Programming

I have a strong command of the Functional Programming Paradigm. Considered esoteric by many, it's been finding its way into the mainstream lately. Functional style emphasizes minimizing mutable state and using functions as first-class objects. This tends to minimize programming errors and allows programs to be reasoned about mathematically. I'm comfortable with both the Lisp family of languages and strongly-typed type-inferred languages descended from Miranda and ML, such as Haskell.

Object Oriented Programming

While I prefer the Functional Paradigm, I'm no slouch at doing things the OO way. I have a solid command of all of the standard design patterns. My style is influenced by the time I spent programming smalltalk, so I prefer dynamicly typed OOP languages. When programming in an OOP style, I tend to focus more on message passing than inheritance.
